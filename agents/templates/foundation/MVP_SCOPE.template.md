## 4. Anti-overengineering checklist
- [ ] No microservices unless one is unavoidable and justified in `RISKS_AND_ASSUMPTIONS.md`
- [ ] No multi-region, multi-tenant, or sharding decisions in v1 unless required by criticality
- [ ] No queue, event bus, or async pipeline unless a synchronous path cannot satisfy the use case
- [ ] No auth provider beyond what's needed for the MVP user base
- [ ] No analytics / observability infrastructure beyond minimal logging in v1
- [ ] No abstractions for variations that don't yet exist
- [ ] No code generation, plugin systems, or framework-style flexibility in v1

If any box is unchecked, the corresponding item must be justified explicitly.
